## 🌟 About The Project

This is a complete landing page for a headphones company. It was built from a Figma design and includes everything a modern website needs - from a cool header to a working contact form!

I built this step by step while learning frontend development at ALX. Each section taught me something new about HTML, CSS, and JavaScript.

## ✨ Features

### 1. **Header Section**

- Eye-catching background image
- Navigation menu that works on all devices
- Big "Call to Action" button
- Hero text that explains the brand

### 2. **"What We Do" Section**

- Four feature cards with icons
- Clean and simple design
- Icons from custom font (holberton_school-icon)
- Hover animations on each card

### 3. **"Our Results" Section**

- Dark background that stands out
- Four pentagon shapes with statistics
- Shows +2% growth and other metrics
- Hover effects on each pentagon

### 4. **Contact Form**

- Name, Email, and Message fields
- Beautiful underline style inputs
- Validation (can't submit empty fields)
- Same "Call to Action" button style

### 5. **Footer**

- Company logo
- Social media links (Facebook, Twitter, Instagram)
- Copyright information
- Clean dark background

### 6. **Mobile Menu (Hamburger)**

- ☰ icon appears on small screens
- Click to open/close menu
- Smooth slide-down animation
- Closes when you click a link or outside

### 7. **Responsive Design**

Works perfectly on:

- 💻 Desktop (4 columns)
- 📱 Tablet (2 columns)
- 📱 Mobile (1 column)

### 8. **Fun Animations**

- Cards scale up when you hover over them
- Icons change color
- Pentagons bounce slightly
- Smooth transitions everywhere

## 🛠️ Technologies Used

- **HTML5** - For the page structure
- **CSS3** - For styling and animations
- **JavaScript** - For the mobile menu
- **Google Fonts** - For nice typography
- **Figma** - For design reference

## 📚 What I Learned

### HTML Concepts:

- How to structure a webpage with `<header>`, `<section>`, `<footer>`
- Creating forms with `<input>` and `<textarea>`
- Using custom icon fonts
- Making websites accessible with ARIA labels

### CSS Concepts:

- **Flexbox** - For arranging items in rows and columns
- **CSS Grid** - For tablet layouts (2 columns)
- **Media Queries** - Making the site work on all screens
- **Variables** - Using `:root` for colors (--main-color, etc.)
- **Animations** - Adding life with `@keyframes` and `transition`
- **Clip-path** - Creating pentagon shapes with CSS
- **Positioning** - Using `absolute` and `relative` to layer elements

### JavaScript Concepts:

- **DOM Manipulation** - Adding/removing classes
- **Event Listeners** - Responding to clicks
- **Toggle Function** - Opening and closing the menu
- **Accessibility** - Managing focus and ARIA attributes
